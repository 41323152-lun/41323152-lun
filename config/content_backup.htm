W3

用什麼套件完成 5-bar Linkage Plotter 與 Openduck mini 零組件設計繪圖？ Solvespace, Siemens NX, Onshape 或其他？各有什麼優缺點？ (plotter.7z, open_duck_mini.7z)

我們組想使用 Onshape 作為主要的 CAD 工具，因為它的雲端協作特性非常符合我們團隊分散、需要同時編輯與版本控管的需求。相較於 SolveSpace 的輕量與 NX 的全面強大，Onshape 在功能與便利性之間取得了平衡，不需要安裝或維護軟體，任何成員只要有瀏覽器就能即時參與設計。我們認為這不僅能提升效率，也能讓溝通更順暢，更適合我們專案的合作模式。

功能 / 面向 SolveSpace Onshape Siemens NX 備註 / 備選工具
授權 / 成本 免費 / 開源 訂閱制 / 商業授權 高價商業授權 對小型／個人用戶：SolveSpace 或中階方案較合適
安裝 / 環境 本地執行、跨平台 雲端 (無需安裝端程式) 本地 + 可能有伺服器 / 客戶端配置 雲端 vs 本地的選擇是關鍵因素之一
協作 / 版本管理 基本支援（有限） 強／即時協作、版本控制 較弱（主要依賴 PLM 整合） 若團隊合作需求高，Onshape 有明顯優勢
模型 / 曲面能力 中等（較擅長簡單實體 / 機構） 中高 極高（複雜曲面 / 自由形狀） 若你做曲面設計、外形設計，NX 比較有優勢
約束 / 參數設計 強 有支援 強（但可能更複雜） SolveSpace 在參數設計理念上較純粹
組合體 / 裝配 可做簡單裝配 支援多零件裝配 支援大型複雜裝配 NX 能處理複雜結構大型機械
模擬 / 數值分析 無或非常有限 基本模擬或需額外模組 完整 CAE / FEA / 流體 / 多物理場 如果設計過程需要密集模擬，NX 優勢明顯
製造 / CAM 整合 需依賴外部工具 有部分 CAM 模組或介面 本身整合 CAM 模組 NX 在 CAM / 製造端整合強
使用者界面 / 學習曲線 偏技術、介面普通 現代、較友善 複雜、學習門檻高 新手可能先從 Onshape 或 SolveSpace 入門
穩定性 / 對大模型能力 弱於大模型 中等 強於大型專案 若要做大型專案或超大裝配，NX 比較穩健

W7

一、零組件設計完成後，如何「切割零件」？ 這一步其實就是： 👉 把 CAD 檔變成真的零件 5-bar Plotter 最常見： 連桿：雷射切割 or 3D 列印 基座：雷射切割 筆座：3D 列印 2️⃣ CAD → 切割檔流程（實務一定要這樣） 在 CAD 中「拆零件」 每一根連桿 每一個基座 不要整台一起輸出 ❌

二、為什麼一定要「切割零件」？

為了要「工程化」

三、如何利用 Webots 進行後續控制系統設計？ Webots 在這裡的角色：實體前的數位分身 1️⃣ 建立「對應實體」的 Webots 模型 2️⃣ 動態模擬（Dynamic Simulation）是什麼？ 不是只有「角度對」而已，而是： 轉動慣量 摩擦 重力 連桿擺動 3️⃣ 控制系統設計流程（標準工程版）

W10

一、Webots 是什麼？

Webots 是一套機器人模擬軟體 可以在電腦裡「先模擬機器人」，不用真的做出來，就能測試： 機器人怎麼走 馬達怎麼轉 感測器怎麼讀 控制程式對不對 👉 常用在： 機器人課程 專題製作 控制系統設計 強化學習、AI 機器人

二、Webots 有什麼特點？ 1️⃣ 3D 真實物理模擬 有 重力、摩擦力、碰撞 用 ODE / Bullet 物理引擎 機器人跌倒、卡住都會真的發生 2️⃣ 內建很多機器人 不用自己從零畫： e-puck NAO TurtleBot 工業手臂 也可以自己設計機器人模型 3️⃣ 支援多種程式語言 最常用的是： ✅ Python（最適合初學 & 教學） C / C++ Java MATLAB（外接） 4️⃣ 感測器很完整 模擬很真實： 距離感測器 相機 IMU GPS 編碼器 Pen（你之前用來畫線的） 5️⃣ 適合控制系統與演算法測試 可以做： PID 控制 路徑規劃 逆運動學（IK） 強化學習訓練

三、如何進行場景模擬？（基本流程） Step 1️⃣ 建立世界（World） 開 Webots New Project 建立 .wbt 世界檔 世界裡面可以放： 地板 牆壁 障礙物 機器人 Step 2️⃣ 放入機器人 拖內建機器人 或載入自己做的機器人（PROTO） 設定： 馬達 感測器 控制器（Controller） Step 3️⃣ 設定控制器4️⃣ 啟動模擬

四、如何用 Python 進行控制系統設計？ 1️⃣ 基本 Python 控制架構 2️⃣ 控制馬達 3️⃣ 讀取感測器（例：距離感測器） 4️⃣ 控制系統設計

W14

一、Webots 的模型能不能用「程式方法」建立？ 可以，而且是 Webots 的核心設計之一 Webots 的所有世界與機器人，本質上都是用一種文字描述語言： VRML / Webots Scene Tree（.wbt / .proto）1️⃣ 直接用文字描述建立模型 2️⃣ 用 PROTO 3️⃣ 用 Python 在「模擬時期」動態建立（進階）

二、能不能從 CAD 套件轉出 Webots 模型？ 可以，而且是業界常見流程 但要注意： CAD ≠ 機器人模擬模型（需轉換） 1️⃣ Webots 支援的 CAD 匯入格式 2️⃣ 正確的 CAD → Webots 流程 4️⃣ 機器人 CAD → Webots
